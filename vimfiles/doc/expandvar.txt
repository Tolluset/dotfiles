*expandvar.txt*	Plugin for expanding identifiers in buffer
                                                Last change: 2010 Jan 05.

SYNOPSIS						*expandvar-synopsis*
>
	" in a buffer tied to "example.txt"
	v:lang
	&background
	'%:t'
	printf('*%s*',expand('%:t:r'))

	" hit <Leader>ev on each identifier for expanding
	ja
	dark
	example.txt
	

	" hit <Leader>ee on each identifier for evaluating
	ja
	dark
	%:t
	*example*

DESCRIPTION					*expandvar-description*

Plugin for expanding identifiers (variables, options and so on) and evaluating
expressions in the current buffer.  Both features target <cWORD>.

DEPENDENCIES					*expandvar-dependencies*

This plugin require following autoload file.

autoload/jwlib/buf/replace.vim
http://github.com/januswel/dotfiles/blob/master/vimfiles/autoload/jwlib/buf/replace.vim

PROVIDES						*expandvar-provides*

Mappings:
<Leader>ev or <Plug>ExpandVariable
	Expand identifiers.

<Leader>ee or <Plug>EvalExpression
	Evaluate expressions.

SETTINGS						*expandvar-settings*

DISABLES

Variables:
g:loaded_expandvar
	When exists, this plugin do nothing.  Use to disable this plugin.

g:no_expandvar_maps
	When non-zero, this plugin doesn't register any effective mappings.

g:no_plugin_maps
	When non-zero, this plugin doesn't register any effective mappings.
	This variable may have an affect on mappings of other plugins.  Use
	"g:no_expandvar_maps" to disable only mappings of this plugin.

KOWN ISSUES					*expandvar-knownissues*

For several option with top of '&' (see |:let-&|), may be expanded to an
irrelevant value.  E.g. "&statusline", "&tabline" etc.  When an option isn't
expanded correctly, use evaluating instead.  This is the issue of VIM itself,
not the other way around the one of this plugin.
